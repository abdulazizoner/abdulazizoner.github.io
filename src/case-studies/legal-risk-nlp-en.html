Prototype

Turkish Legal-Risk Text Classification

A decision-support prototype for analyzing public Turkish social media text — designed to assist human reviewers with NLP preprocessing and signal highlighting, not to automate legal enforcement or provide legal advice.

Overview

This project explores how NLP techniques can support — not replace — human judgment when reviewing public Turkish social media content for potential legal-risk signals. The motivation is educational and technical: understanding preprocessing, classification, and keyword matching in a sensitive domain where accuracy errors have real consequences.

The prototype is explicitly framed as human-in-the-loop decision support. It does not report content to authorities, does not make binding legal determinations, and does not provide legal advice. Any flagged output is meant for a person to review, contextualize, and decide upon. KVKK (Turkey's personal data protection law) and GDPR principles inform how I think about data scope, minimization, and purpose limitation — even in a student prototype.

I am building this to learn responsible NLP design in a high-stakes context. The system is not deployed, not connected to live moderation pipelines, and not validated for legal accuracy. It is an experimental tool for studying pipeline design and ethical constraints.

Problem

Public Turkish social media text may contain language that warrants careful review for legal-risk signals, but fully automated classification is risky. Models can produce false positives on sarcasm, quoted speech, or ambiguous phrasing. They can miss context that a human reader would catch. Automating enforcement decisions raises ethical concerns around privacy, due process, and overreach. A more responsible starting point is tooling that helps a reviewer prioritize and understand text — with a human always in the loop.

Goals

  • Build a Turkish text preprocessing pipeline suitable for social media-style input
  • Experiment with classification approaches that surface legal-risk signals as review hints
  • Implement keyword and legal-concept matching to complement model outputs
  • Design reviewer-facing output that presents signals clearly without claiming certainty
  • Keep human review mandatory — no automated reporting or authority integration
  • Document ethical constraints, KVKK/GDPR awareness, and known failure modes

System / workflow

Public text input (sample / test dataset) → Turkish normalization (lowercasing, punctuation, token cleanup) → Preprocessing & feature extraction → Classification model / rule layer (risk signal scores) → Keyword & legal-concept pattern matching → Aggregated review output (flags, scores, matched terms) → Human reviewer reads original text + system hints → Human makes final judgment (accept / reject / escalate manually) → No automated authority reporting

Technical focus

  • Python NLP preprocessing for Turkish text
  • Text classification experiments (baseline models, feature-based approaches)
  • Rule-based and model-assisted keyword/concept matching
  • Sentiment analysis as a supplementary signal — not a sole decision factor
  • Structured output format for reviewer-facing results
  • Documentation of false positive patterns and evaluation limitations

What I am building

  • Turkish text normalization and cleaning utilities for short-form social content
  • Classification pipeline that outputs risk signal scores with confidence caveats
  • Keyword and legal-concept matcher to highlight terms worth manual inspection
  • Prototype review interface or structured JSON/text output for human evaluation
  • Test notebooks documenting preprocessing choices and classification experiments
  • Ethics and limitations documentation (no legal advice, no authority claims)

Current state

The project is a prototype under active development. It has not been deployed in any official, commercial, or enforcement context. There are no claims of real-world usage, validated legal accuracy, or integration with authorities or platforms. All outputs are intended for experimental and educational review on controlled sample data. I do not present this as a finished moderation product.

Limitations and risk awareness

False positives are expected: sarcasm, jokes, quoted material, and political rhetoric can trigger signals that a human would dismiss. Legal ambiguity is inherent — NLP models do not interpret law and cannot account for full situational context. The prototype does not handle dialect variation, code-switching, or multimedia context (images, threads) robustly. This is not legal advice, not a compliance tool, and not suitable for automated enforcement. Any use beyond learning would require legal review, domain expert validation, and formal governance — none of which this student project provides.

Next steps

  • Expand Turkish preprocessing handling for informal spelling and social abbreviations
  • Build a small labeled evaluation set and track precision/recall on known examples
  • Document false positive cases (sarcasm, quotes) and adjust rules where possible
  • Improve reviewer output layout to separate signals from original text clearly
  • Write a clearer ethics section covering KVKK/GDPR data scope assumptions
  • Explore human-in-the-loop evaluation protocol before any broader testing
